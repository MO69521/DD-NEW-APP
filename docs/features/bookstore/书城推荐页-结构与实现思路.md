## 1. 页面定位

书城推荐页是 App 的**主 Tab 首页**（底部导航「书城」为默认选中态）。页面职责：

- 提供搜索与分类入口
- 展示多维度榜单（推荐 / 人气 / 飙升 / 完结）
- 展示编辑推荐与个性化推荐
- 承载底部 5 Tab 主导航

当前实现阶段：**UI Only 静态原型**（mock 数据，无 API / 无路由跳转）。

## 2. 整体布局结构

页面采用 **「固定顶栏 + 可滚动内容 + 固定底栏」** 三层结构：

```
书城推荐页 (142:1828)  375×2086
├── 状态栏 (173:1654)                    ← 系统层，Flutter 不实现
├── 搜索栏区 (142:1953)                  ← 固定，y=44，高 44
│   ├── 搜索框 (142:1954)
│   └── 分类按钮 (142:1960)
├── 滚动内容区 (142:1829)                ← y=108 起，宽 351，左右各 12px 边距
│   ├── 推荐榜卡片 (142:1992)            ← 间距 gap 32px
│   ├── 编辑推荐 (142:1877)
│   └── 猜你喜欢 (142:1917)
└── 底部导航 (173:1735 → 184:2116)       ← 固定底部，高 82
    ├── 5 Tab 胶囊导航
    └── Home Indicator (184:2189)
```

### 2.1 滚动策略

| 区域 | 滚动行为 | 说明 |
|------|----------|------|
| 搜索栏 | 随内容滚出 / 或吸顶 | 设计稿为 absolute 固定；实现用 `SliverSafeArea` + 顶部 header |
| 内容区 | `CustomScrollView` 纵向滚动 | 三个区块纵向堆叠，区块间距 32px |
| 底部导航 | 固定 `Positioned(bottom: 0)` | 渐变遮罩 + 胶囊 Tab，内容区需预留底部 padding |

### 2.2 区块间距（Figma → Token）

| 设计值 | Token | 用途 |
|--------|-------|------|
| 12px | `AppSpacing.sm` | 页面左右边距、搜索栏 horizontal padding |
| 16px | `AppSpacing.md` | 卡片内 padding、行/列间距、网格 gap |
| 24px | `AppSpacing.lg` | 区块标题与内容间距 |
| 32px | `AppSpacing.xl` | 大区块之间间距 |
| 8px | `AppSpacing.xs` | 封面与文字、Tab 指示器间距 |
| 4px | `AppSpacing.xxs` | 图标与文字、导航内 padding |

## 5. 组件分级与目录映射

遵循项目 Feature First 架构（见 `.cursor/rules/flutter-architecture-strict-v2.mdc`）：

```
lib/
├── core/theme/                          ← 设计 Token（颜色/间距/圆角/文字/动画）
├── shared/
│   ├── widgets/                         ← L1: AppText, BookCover, AppButton
│   └── components/                      ← L2: 跨 feature 复用
│       ├── GlassChipButton
│       ├── SectionHeader
│       ├── BookListTile                 ← 榜单横向书项
│       └── BookGridCard                 ← 网格竖向书项
└── features/bookstore/
    ├── domain/entities/book.dart        ← 领域实体
    ├── data/                            ← 后续 API / 缓存
    ├── application/                     ← 后续 Cubit + State
    └── presentation/
        ├── pages/bookstore_page.dart    ← 页面组装
        └── components/                  ← L3: feature 专属

            ├── editor_pick_section.dart
```

### 5.1 复用判断

| 组件 | 级别 | 理由 |
|------|------|------|
| `BookCover` | L1 | 全局书籍封面 primitive |
| `BookListTile` / `BookGridCard` | L2 | 已在榜单/推荐/猜你喜欢复用，≥3 次 |
| `SectionHeader` | L2 | 多区块通用标题 pattern |
| `GlassChipButton` | L2 | 搜索/分类玻璃态容器 |
| `RankingSection` 等 | L3 | 书城页专属布局组合 |

## 7. 状态管理规划

按架构规则，state 只在 `application` 层创建，分三类：

### 7.1 UI State

- 页面 loading / error / empty
- 下拉刷新状态
- 骨架屏

### 7.2 Domain State

- 当前选中榜单 Tab → `RankingTab` enum
- 各区块书籍列表 `List<Book>`
- 搜索热词 placeholder

### 7.3 Interaction State

- 底部 Tab 选中 index（最终上提到 Shell）
- 榜单 Tab 切换动画（当前在 `RankingSection` 内部 StatefulWidget，后续迁入 Cubit）

### 7.4 Cubit 职责草案

```
BookstoreCubit
├── load()                    并行加载三区块数据
├── switchRankingTab(tab)     切换榜单并请求
├── refresh()                 下拉刷新
└── onBookTap(bookId)         触发路由（通过 callback 或 Router delegate）
```

## 10. 性能要点

- 列表必须使用 `GridView.builder` / `SliverChildBuilderDelegate`（当前区块数据量小，已用 builder）
- 嵌套滚动：`shrinkWrap: true` + `NeverScrollableScrollPhysics()`（区块内网格不参与外层滚动竞争）
- 底部固定栏：内容区预留 `BookstoreBottomNav.barHeight + AppSpacing.xl`
- 大列表阶段：猜你喜欢改 `SliverGrid` + 分页，避免一次 build 全部 item
- 封面图：后续网络图需缓存 + 占位 + 错误图
