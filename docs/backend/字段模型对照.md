# 字段模型对照

本文把前端核心 domain model 翻译成后端字段说明。字段命名可按后端规范调整，但含义需要保持一致。

## 通用字段约定

| 前端当前字段 | 后端建议字段 | 类型 | 说明 |
|---|---|---|---|
| `id` | `id` | string | 全局唯一 ID，前端用于跳转和列表 key |
| `coverAsset` | `coverUrl` | string | 当前 mock 是本地资源，真实接口建议 URL |
| `avatarAsset` | `avatarUrl` | string | 当前 mock 是本地资源，真实接口建议 URL |
| `backgroundAsset` | `backgroundUrl` | string | 当前 mock 是本地资源，真实接口建议 URL |
| `timeLabel` | `createdAt` / `updatedAt` | string/date | 前端可格式化为“昨天”“4天前”等 |
| `xxxText` | 原始字段或展示文案 | string | 如果文案强运营，可后端返回；如果是数字格式，建议返回原始值 |

## 展示字符串与原始值

当前 mock 为方便 UI，很多字段已经是展示字符串。真实接口建议尽量返回原始值，由前端格式化：

| 展示字段 | 建议原始字段 |
|---|---|
| `12.8万粉丝` | `followerCount: 128000` |
| `235.6` 热度 | `popularity: 235600` 或业务定义数字 |
| `4天前` | `createdAt` / `updatedAt` |
| `23 分钟` | `todayReadingMinutes: 23` |
| `9.9` | `priceYuan: 9.9` 或 `priceCents: 990` |

如果后端已有统一格式化规范，也可以直接返回展示文案，但需要字段名明确为 `displayText` / `label`。
