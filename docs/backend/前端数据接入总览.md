# 前端数据接入总览

本文按功能模块整理当前前端数据来源。后端同事可以通过本页快速定位：页面需要什么数据、mock 在哪里、未来要替换哪个 data source。

## 读法

- `Cubit`：页面状态和交互逻辑入口。
- `Repository`：domain 层数据契约，前端业务只认这个抽象。
- `MockDataSource`：当前静态数据位置，真实接口接入时优先替换这里。
- `Domain Model`：页面最终使用的数据形态，后端响应建议映射到这些模型。

## 全局基础能力

| 能力 | 当前文件 | 说明 |
|---|---|---|
| 服务定位 | `lib/core/services/service_locator.dart` | 当前所有全局服务入口，后续可集中挂真实服务 |
| 登录抽象 | `lib/core/services/auth_service.dart` | 登录、验证码、一键登录、退出登录 |
| 登录 REST 占位 | `lib/core/services/rest_auth_service.dart` | 已写出路径草案，但 `_post` 仍抛错 |
| 登录会话 | `lib/core/services/auth_session_service.dart` | 当前内存 session，后续建议持久化到安全存储 |
| 当前用户/VIP | `lib/core/services/membership_status_service.dart` | Profile、福利、会员等页面共享用户/VIP 状态 |
| 书架共享状态 | `lib/core/services/bookshelf_membership_service.dart` | 当前本地维护已加入书架的书 |

## 当前交互动作状态

| 动作 | 当前状态 | 后端接入建议 |
|---|---|---|
| 登录/验证码 | `RestAuthService` 有路径占位，尚未接 HTTP | 优先接入，作为用户态接口前置 |
| 加入/移除书架 | `BookshelfMembershipService` 本地维护 | 接 `POST/DELETE /me/bookshelf` |
| 猜你喜欢加载更多 | 前端复制 mock seed | 接分页列表，返回 `cursor` 或 `page` |
| 搜索历史 | 当前 mock/local | 登录用户可后端保存，未登录可本地保存 |
| 签到 | `WelfareCubit.checkIn()` 本地切状态 | 接签到写接口，返回奖励和当天状态 |
| 送心/点赞/评论 | 书籍详情内本地模拟 | 接写接口，并返回更新后的统计 |
| 会员开通 | 本地更新 `MembershipStatusService` | 接支付/会员订单接口 |
| 伙伴筛选/排序 | 前端对 mock 列表过滤 | 数据量大后建议后端分页筛选 |

## 后端接入替换点

后端实现接口后，前端建议按以下方式替换：

1. 保留 `domain/entities` 和 `domain/repositories`。
2. 在 `features/<name>/data/datasources/` 新增 `*_api_datasource.dart`。
3. 新增 DTO 文件，例如 `features/bookstore/data/dtos/bookstore_dto.dart`。
4. `RepositoryImpl` 负责调用 datasource 并把 DTO 映射到 domain entity。
5. 通过构造参数或 service locator 切换 mock/api。

示例：

```text
features/bookstore/
  domain/repositories/bookstore_repository.dart   # 不变
  data/datasources/bookstore_mock_datasource.dart # 保留给预览
  data/datasources/bookstore_api_datasource.dart  # 新增
  data/dtos/bookstore_dto.dart                    # 新增
  data/repositories/bookstore_repository_impl.dart # 选择 datasource + mapper
```
